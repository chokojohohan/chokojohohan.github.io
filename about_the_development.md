## 3. 要件定義

### 3.1. 機能要件
- **静的サイト**: GitHub Pagesを利用した静的ホスティング。
- **レスポンシブデザイン**: PC、タブレット、スマートフォンに対応。
- **アニメーション**: スクロール連動アニメーション、ローディング画面、ハンバーガーメニュー。

### 3.2. 非機能要件
- **パフォーマンス**: 画像の軽量化（WEBP使用）による読み込み速度の最適化。
- **アクセシビリティ**: 適切なコントラスト比、alt属性の記述。

## 4. デザイン定義

### 4.1. カラーパレット
- **メインカラー**: `#4372c0` (青系)
- **ブラック**: `#000000` (文字色、黒背景セクション)
- **ホワイト**: `#ffffff` (背景色、白背景セクション)

### 4.2. フォント
- **メインフォント**: `Noto Sans JP` (日本語、英数字等)
- **アクセントフォント**: `Makinas-Flat` (タイトル等), `Comfortaa` (KawaiiLogo著作権表示)

### 4.3. レイアウト
- **ヘッダー**: 固定ヘッダー（スクロールで追従）。
- **ナビゲーション**: ハンバーガーメニューによる全画面展開。
- **セクション構成**: 白背景と黒背景のセクションを交互に配置し、メリハリをつける。

### 4.4. 画像・アイコンの取り扱いに関する注意点
- **ファイル形式**: **SVG以外は全てWEBP形式に変換して使用すること。**
- **WEBP変換方法**:
    - [Squoosh](https://squoosh.app/) を使用してWEBP形式に変換してください。（「WEBP v2 (unstable)」は使用しない。）
    - 画質を保ちつつ、ファイルサイズを可能な限り小さくすることができます。
- **動画の取り扱い**:
    - GitHubのアップロード制限があるため、動画ファイルは直接アップロードせず、**YouTubeに投稿**してください。
    - 情報工学班のGmailアカウントにログインし、YouTubeにアクセスしてください。
    - YouTubeの埋め込みコード（iframe）を使用して、動画を掲載してください。
    - `about.html`の「情報工学班について」セクション内の動画は、YouTubeの禁止されている中学生のアカウント等でも閲覧を可能にするため、画質を下げたものをYouTube埋め込みとは別に掲載してください。

## 8. サイト刷新の手順

将来的にサイトを全面的に刷新（リニューアル）する場合の手順です。

1. **現行リポジトリのアーカイブ**
    - 現在の `chokojohohan.github.io` リポジトリの名前を変更します（例: `chokojohohan.github.io-v1` や `archive-2025` など）。
    - これにより、旧サイトのコードをバックアップとして保存します。

2. **新規リポジトリの作成**
    - 新しく `chokojohohan.github.io` という名前でリポジトリを作成します。
    - これが新サイトの公開用リポジトリとなります。

3. **コードの追加**
    - 新しいリポジトリに、新サイトのソースコードを追加・プッシュします。

4. **公開設定**
    - GitHub Pagesの設定を行い、新サイトを公開します。

5. **検索エンジンへの対応**
    - Google Search Console や Bing Webmaster Tools にアクセスし、サイトの再検証やサイトマップの送信を行い、新サイトが正しくインデックスされるようにします。
